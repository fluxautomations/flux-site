Case studies

Case study — 01
Accounts ReceivableTally XMLOutlook

Intelligent AR & follow-up engine

The problem

Tracking aging bills, matching them against client ledgers, and manually sending follow-up emails was time-consuming and prone to human error.

The solution

An automated pipeline extracts real-time "Bills Receivable" data from Tally, builds aging buckets, generates an interactive Excel dashboard with forecast charts, and dispatches personalized follow-up emails via Outlook.

Case study — 02
AI / MLTelegram BotOCR

AI-powered invoice processor bot

The problem

Processing vendor invoices from WhatsApp, Telegram or email meant opening PDFs, reading unstructured formats, and typing data into an Excel log by hand.

The solution

A custom Telegram bot powered by smart AI OCR: forward an invoice image or PDF, and within seconds it extracts vendor, date, GST amount, category and total — logging it into a master sheet and posting the entry to your ERP on command.

Case study — 03
Streamlit AppAccounts PayableBank Integration

End-to-end automated payouts dashboard

The problem

Managing vendor payouts meant checking Tally for outstanding bills, formatting bank upload files (NEFT/RTGS), and manually emailing payment advice to every vendor.

The solution

A secure local web dashboard ingests payout inputs, uses AI fuzzy-matching to find the right vendor, allocates payments to outstanding bills on a FIFO basis, pushes the transaction back to your ERP, generates bank-ready files, and bulk-sends payment advice emails.

Case study — 05
Browser AutomationsPlaywrightNo API Needed

Browser automation without API access

The problem

Many portals — government sites, vendor dashboards, legacy ERP interfaces — offer no API. Repetitive tasks like downloading reports, submitting forms, or checking order statuses had to be done manually, browser tab by browser tab.

The solution

Using Playwright and Selenium, we automate the browser itself — logging in, navigating pages, filling forms, clicking buttons, and extracting data exactly as a human would, but on a schedule and at scale. If you can do it in a browser, we can automate it.
